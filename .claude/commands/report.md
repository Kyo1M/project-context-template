# /report — 定例報告メモ生成スキル

定例会議の報告メモのドラフトを自動生成します。前回報告メモ・wiki・議事録・メモから情報を収集し、構造化された報告メモを作成します。

## 使い方

```
/report {定例名}
```

例:
```
/report VGB定例
/report DS定例
```

## 処理内容

### 1. 情報を収集する

以下の4つのソースから情報を収集する。

#### 1a. 前回の報告メモ

- `docs/deliverables/` 内の報告メモ（ファイル名に定例名のスラッグを含むもの）から最新ファイルを特定する
- 「ネクストアクション」テーブルの内容を取得する
- 前回の報告メモが存在しない場合（初回実行時）はスキップする

#### 1b. wiki プロジェクト状況

- `docs/wiki/index.md` を読む
- プロジェクトの現在の状況、既知の課題・リスク情報を収集する

#### 1c. 議事録

- `docs/minutes/` から、前回報告メモの日付以降に作成された議事録を特定する
- 各議事録の「議論サマリー」「決定事項」「次のアクション」を収集する
- 前回報告メモがない場合は、直近2週間の議事録を対象とする

#### 1d. メモ

- `docs/memo/` から、前回報告メモの日付以降に作成されたメモを特定する
- 各メモの概要を収集する

### 2. 報告メモを生成する

収集した情報を以下のフォーマットで整形し、ファイルに書き出す。

#### 出力先

`docs/deliverables/yyyymmdd_{定例名スラッグ}-report.md`

（yyyymmdd は今日の日付）

#### 出力フォーマット

```markdown
---
type: deliverable
title: {プロジェクト名} 定例報告 — YYYY-MM-DD
date: YYYY-MM-DD
status: draft
audience: internal
format: report
topics: []
tags: []
derived_from: []
related: []
---

# {プロジェクト名} 定例報告 — YYYY-MM-DD

## 前回の振り返り

| アクション | 担当 | 状況 |
|-----------|------|------|
| （前回ネクストアクションから展開） | | 未確認 |

## 報告事項

- （議事録・メモ・wikiから収集した実施内容・共有事項）

## 課題・リスク

| 課題 | 影響 | 対応方針 |
|------|------|---------|
| （wikiの既知の課題から収集） | | |

## 議論事項

<!-- 定例で議論・判断が必要な事項を記入してください -->
- [ ] 

## ネクストアクション

<!-- 定例後に記入してください -->
| アクション | 担当 | 期限 |
|-----------|------|------|
| | | |
```

### 3. ユーザーに案内する

生成後、以下を案内する：
- 生成したファイルのパスを表示する
- 「議論事項」セクションに定例で議論したい内容を記入するよう促す
- 「報告事項」「課題・リスク」の内容を確認・修正するよう促す

## 注意事項

- `docs/deliverables/` ディレクトリが存在しない場合は自動で作成する
- 収集した情報をそのまま転記するのではなく、報告に適した簡潔な表現に整形する
- 確認できない情報や推測は含めない
- コミット時はコミットメッセージをユーザーに提示し、承認を得てから実行する

## ワークフロー上の位置づけ

```
/report（本スキル）→ 議論事項記入 → 定例実施 → /minutes → /ingest → タスク実施 → 翌週の /report
```
