# /minutes — 議事録整形スキル

生の議事メモを整形された議事録に変換します。

## 使い方

```
/minutes docs/minutes/_drafts/yyyymmdd_title.md
```

## 処理内容

引数で渡されたファイルパスの生議事メモを読み込み、以下の構造に整形して
`docs/minutes/yyyymmdd_slug.md` として保存してください。

ファイル名のスラッグは英字ケバブケースで、日本語タイトルは frontmatter の `title:` に記載する。

### 出力フォーマット

```markdown
---
type: minutes
title: （日本語タイトル）
date: YYYY-MM-DD
status: active
topics: []
tags: []
attendees: []
meeting_type: sync
derived_from: [（元ファイルパス）]
related: []
---

# 議事録: {タイトル}

**日時**: YYYY年MM月DD日  
**参加者**: （メモから抽出）  
**場所 / 形式**: （メモから抽出、不明な場合は省略）

---

## アジェンダ
（議題を箇条書きで整理）

## 議論サマリー
（各議題の議論内容を簡潔に要約）

## 決定事項
- （決まったことを箇条書き。各項目には背景や理由も1行で添える）

## 次のアクション
| アクション | 担当 | 期限 |
|-----------|------|------|
| （内容） | （担当者名） | （期限 or 未定） |

## 懸案事項・持ち越し
- （未解決の課題、次回に持ち越す議題）

## 備考
（その他、文脈として残しておきたい情報）
```

## 注意事項

- 生メモから読み取れない情報は「（記録なし）」と記載し、推測で埋めない
- 決定事項とアクションは特に正確に抽出する（/ingest で使われる）
- 保存後、生成した議事録のファイルパスを報告する
- コミット時はコミットメッセージをユーザに提示し、承認を得てから実行する
