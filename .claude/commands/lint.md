# /lint — ナレッジヘルスチェックスキル

`docs/` 配下のナレッジ全体をスキャンして健全性を確認・修正します。週次実行を推奨します。

## 使い方

```
/lint
```

## 処理内容

以下の順番でチェックを実行し、結果をレポートしてください。

### チェック項目

#### 1. インデックス整合性チェック
- `docs/wiki/index.md` に記載されているリンク先ファイルが実際に存在するか確認
- `docs/wiki/topics/` 内の全ファイルが `index.md` にリンクされているか確認
- 不整合があれば `index.md` を更新して修正する

#### 2. llms.txt 整合性チェック
- `llms.txt` に記載されている全ファイルパスが実際に存在するか確認
- `docs/` 内のコンテンツファイルが `llms.txt` に記載されているか確認
- 不整合があれば `llms.txt` を更新して修正する

#### 3. frontmatter バリデーション
- `docs/` と `tasks/` 配下の全 `.md` / `.html` ファイルの frontmatter を検証する
- 必須フィールド: `type`, `title`, `date`, `status`
- type の値: `minutes`, `decision`, `memo`, `wiki`, `task`, `deliverable`, `exploration`, `topic`
- status の値: `draft`, `active`, `superseded`, `archived`, `done`, `cancelled`
- date のフォーマット: `YYYY-MM-DD`
- ファイル名パターン: `yyyymmdd_slug.{md,html}`（`index.md` は例外）

#### 4. 孤立記事の検出
- どの記事からもリンクされていない記事を一覧表示する
- 孤立記事が見つかった場合は、関連しそうなトピックハブからリンクを追加する提案をする

#### 5. 欠損記事の検出
- 複数の記事で言及されているが、それ自体の記事がないトピックを検出する
- 欠損記事の候補リストを表示する（自動作成はしない、要人間判断）

#### 6. 矛盾・古い情報の検出
- 同じトピックについて異なる記事が矛盾する記述をしていないか確認する
- `[要確認]` タグがついている箇所を一覧表示する

### 出力レポートのフォーマット

```
## /lint レポート — YYYY-MM-DD

### インデックス整合性
- ✅ 問題なし / ⚠️ {件数}件の不整合を修正

### llms.txt 整合性
- ✅ 問題なし / ⚠️ {件数}件の不整合を修正

### frontmatter バリデーション
- ✅ 全ファイル通過 / ⚠️ {件数}件の問題
  - （問題のあるファイル名と内容）

### 孤立記事
- （ファイル名一覧）

### 欠損記事候補
- （トピック名一覧）

### 矛盾・要確認
- （記事名と該当箇所の一覧）

### 次のアクション推奨
- （人間が対応すべき事項の優先順リスト）
```
